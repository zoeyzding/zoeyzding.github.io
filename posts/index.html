五月花号后代的数量真的有3500万吗？

1620年9月，102位乘客乘坐五月花号轮船(Mayflower)从英国启程前往北美。到达目的地的时候，只有53人在第一个冬天存活下来，而其中只有5名成年女性。如今，统计到的全球范围内的五月花号乘客的后代数量大约是3500万。所以这个数据可靠吗？ 假设某五月花号后代和一名非五月花号后代结婚， 并且生了两个孩子，他们的下一代里， 五月花号后代的数量将会翻番。如果这一代人里，仍旧是一名五月花号后代和一名非五月花号后代结婚并生两个孩子，那么再下一代里五月花号后代数量还会继续翻番。 假设一个五月花号后代和另一个五月花号后代结婚， 并且有两个孩子。这种情况下，这两个孩子只是取代了他们的父母。 从整个群体来看，五月花号后代数量并不会增加。 另外， 考虑到血统崩溃 (Pedigree collapse)带来的影响， 五月花号后代的数量要比上述的假设要少。 数学家Rob Esterway做了一个统计分析，五月花号乘客之一的Richard Warren的后代数量差不多就有200万人。 所以全球范围内有3500万五月花号后代也是正常的。 链接 Maths and the Mayflower (BBC Channel 4: More or Less)...

ESLint, Prettier & Beautify: VSCode Formatting

问题 VSCode代码自动格式化使用的主流插件是ESLint，Prettier和Beautify。其中ESLint需要通过Terminal安装，后面两个是VSCode的插件，三个插件互相配合可以对代码排版起到事半功倍的效果。 最近写React代码遇到了的格式上的问题，即代码自动格式化之后换行较多，导致代码看起来不够清爽。 于是研究了一下这三个插件的设置。 __ ESLint 首先通过npm i -g eslint安装ESLint，之后在settings.json文件中配置如下的设置： { "eslint.autoFixOnSave": true, "files.eol": "\n" } __ Prettier 在VSCode应用市场安装后做如下配置: { "prettier.eslintIntegration": true, "editor.formatOnSave": true } __ Beautify 同样是在VSCode应用市场安装后做如下配置: { "beautify.language": { "html": ["html", "php", "erb"], "css": [], "js": [] } } — 但是到这里还没有结束。试了一下还是会出现很多不必要自动换行，主要是HTML标签。所以这里还需要对HTML进行unformatted的设置。 //设置不需要自动格式化的html标签类型 "html.format.unformatted": "pre,code,textarea,li", //或者直接禁用所有对html的格式化 "html.format.enable": false 到这里整体代码可以自动换行，但不会过于破碎影响阅读。 — 补充 编译器UI的美观程度也是影响写代码的重要因素。一个简洁美观的界面会让人更有动力写代码。日常使用VSCode的时候，我个人还会有如下的习惯： VSCode主题：City Lights theme 代码字体：FiraCode 机械键盘增强音效插件：Mechvibes — Reference The 2019 Guide to Beautifying Visual Studio Code prevent-new-line-in-vs-code-after-formatting https://blog....
